Dev Lens Protocol

(сокращается до DLP, dlp, dev-lens, или dev-lens-protocol)

Это единый протокол для общения между всеми средствами разработки.

Вдохновляется:
  • Language Server Protocol - яркий пример того, почему нужны стандарты для тулчейнов
  • Unix pipes - идея утилит как чистых функций, которые могут произвольно компоноваться между собой внешними процессами

Что?

Единый стандарт для всех утилит, которые преобразуют исходный код на любом языке в работающую программу.

Он должен охватывать полный цикл работы с исходным кодом - чтение из файловой системы, получение зависимостей, проверки, преобразование (включая кодген и компиляцию, все её этапы), запись в файловую систему, запуск, инкрементальные обновления, фиксация изменений и прочее...

Rule of thumb - если это делает или может делать IDE или тулчейны, то это оно.

Зачем?

Чтобы изменить всю экосистему инструментов разработки.

Да, мы не мелочимся :)

Инструменты, построенные поверх DLP должны в разы упростить переиспользование кода между разными экосистемами языков, тулчейнов или IDE и упростить их написание.

А ещё нам нужно использовать многопоточность наших компьютеров.
